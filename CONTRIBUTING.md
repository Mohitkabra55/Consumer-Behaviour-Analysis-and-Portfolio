# Contributing Guide

Thank you for your interest in improving this project.

## How to Contribute

1. Fork the repository and create a feature branch.
2. Make focused changes with clear commit messages.
3. Keep the notebook, SQL, and documentation consistent with the project workflow.
4. Test your changes locally before opening a pull request.
5. Submit a pull request with a short summary of the change and the expected impact.

## Contribution Guidelines

- Keep documentation professional and easy to reuse in an academic or portfolio setting.
- Avoid committing temporary files, checkpoints, or local credentials.
- Prefer relative paths over absolute machine-specific paths.
- When updating analysis logic, explain the business reason for the change.
- If you add charts, reports, or screenshots, place them in the recommended project folders.

## Suggested Areas for Improvement

- Notebook cleanup and parameterization.
- SQL query optimization and documentation.
- Dashboard polishing in Power BI.
- Additional exploratory analysis or feature engineering.
- Better project packaging for reproducibility.
